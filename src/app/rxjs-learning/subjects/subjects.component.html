Subject · BehaviorSubject

📡 Component communication — share state & events through a service

Problem: Two components need to talk to each other without a direct parent–child binding. Angular uses a shared service with RxJS Subjects.

What we built: Component A (buttons below) publishes messages. Component B (inbox) and the app Header subscribe through RxjsSharedService.

BehaviorSubject — stores the current notification. Any component that subscribes later still gets the latest value immediately (used for shared state).

Subject — emits one-time action events. Only components subscribed at that moment receive it — no memory (used for fire-and-forget events).
