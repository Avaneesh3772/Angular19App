👥 HR Promotion Shortlisting — find eligible candidates step by step

Problem: HR must shortlist employees for a promotion interview panel — but only from the right department, matching a name search, with 3+ years of service, and only the top 3 candidates.

What we built: A shortlisting dashboard that loads all employees from employeeData.json. The roster starts unfiltered; live search and department filters narrow it. Clicking Build Interview Panel runs filter() + take(3).

shareReplay(1) — loads employee data once; the roster and pipeline share the same cached HTTP response.

debounceTime(600) — search box waits 600ms after you stop typing before filtering the roster.

distinctUntilChanged() — department dropdown skips re-filtering when the same value is selected again.

filter() — keeps only employees with yearsOfService >= 3 from the current visible pool.

take(3) — stops after 3 eligible candidates and builds the final interview panel.

When to use each filtering operator
OperatorWhat it doesIn this HR example
shareReplay(1) Caches the last emission; replays to new subscribers Load employeeData.json once — roster and pipeline share the same data
debounceTime(ms) Waits for a pause before emitting Search box — don't filter on every keystroke, wait until user stops typing
distinctUntilChanged() Skips consecutive duplicate values Department dropdown — re-selecting "Engineering" won't trigger again
filter(predicate) Passes only values matching a condition Keep employees with 3+ years of service for promotion eligibility
take(n) Emits only the first n values, then completes Shortlist only 3 candidates for the interview panel
