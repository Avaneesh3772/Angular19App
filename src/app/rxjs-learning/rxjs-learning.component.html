mergeMap vs concatMap vs switchMap — When to use which?
Operator Concurrency Cancels previous? Use case
mergeMap ✅ Parallel ❌ No Parallel API calls where order doesn't matter
concatMap ❌ Sequential ❌ No (queued) Sequential operations, file uploads in order
switchMap ✅ Latest only ✅ Yes Live search, typeahead, user cancels previous action

}
Key Rule — catchError must return an Observable 
// ✅ Correct — return an Observable (of, EMPTY, throwError)
catchError(err => of([]))           // recover with empty array
catchError(err => EMPTY)            // recover silently (complete without value)
catchError(err => throwError(err))  // rethrow (let parent handle it)

// ❌ Wrong — returning void or a plain value will break the stream
catchError(err => console.log(err)) // This is void, not an Observable!

Subject vs BehaviorSubject — Key Difference
Feature Subject BehaviorSubject
Initial value ❌ None ✅ Required (e.g. new BehaviorSubject(''))
Late subscriber gets ❌ Nothing (misses past emissions) ✅ The last emitted value immediately
getValue() ❌ Not available ✅ Can read current value synchronously
Use case Event bus, fire-and-forget events Shared state, auth status, current user, theme
