📦 E-Commerce Order Fulfillment Hub — transform & flatten streams

Problem: An online store must enrich orders, fetch customer data, fulfil shipments in order, and support live order lookup — each task needs a different flattening strategy.

What we built: An order hub loading from ordersData.json. Each accordion demonstrates one transformation operator with a real workflow and visible timing.

map() — transforms each order 1-to-1 (add shipping cost, grand total, ETA). No inner Observable.

mergeMap() — fires 3 JSONPlaceholder user requests in parallel; faster responses arrive first.

concatMap() — runs 3 fulfilment steps per order; the next order waits until the current one finishes all steps.

switchMap() — live order search; typing a new term cancels the previous in-flight lookup.
