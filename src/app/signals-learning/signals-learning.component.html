} 8. mutate() — ⚠️ REMOVED in Angular 17 📝 Why it was removed + how to use update() instead
⚠️ signal.mutate() was available in Angular 16 developer preview but was removed in Angular 17 because it mutated objects in-place, preventing change detection from noticing differences. Use update() with a new object/array instead.

❌ Old way — mutate() (Angular 16, REMOVED)

// mutate() edited the object IN PLACE — Angular
// could not tell the signal had changed because
// the reference was the same object!

items = signal([{{ '{' }}name: 'John', age: 25{{ '}' }}]);

// ❌ This no longer compiles:
items.mutate(arr => {{ '{' }}
  arr[0].age = 30; // mutates in place
{{ '}' }});

✅ New way — update() with spread / map (Angular 17+)

// update() must return a NEW reference so Angular
// knows the signal changed and updates the UI.

items = signal([{{ '{' }}name: 'John', age: 25{{ '}' }}]);

// ✅ Correct — return a new array + new object:
items.update(arr =>
  arr.map(item =>
    item.name === 'John'
      ? {{ '{' }}...item, age: 30{{ '}' }} // NEW object via spread
      : item
  )
);

✅ Adding an item to a signal array

// Always return a NEW array — never push() into the existing one

cartItems.update(items => [
  ...items,               // spread existing
  {{ '{' }}...newProduct, qty: 1{{ '}' }}  // add new item
]);

✅ Removing an item from a signal array

// filter() returns a new array — perfect for signals

cartItems.update(items =>
  items.filter(i => i.id !== removedId)
);
📌 Golden Rule for Signals with Arrays/Objects: Always return a new reference — never mutate the existing value in place. Angular's signal system uses reference equality to detect changes.
